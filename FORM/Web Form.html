Web Form

Introduction

HTML forms are used to collect user input, once a submit button is clicked the input will be sent over the the server "back-end" to process
There are many types of form elements including the following types:

  • checkbox
  • text
  • password
  • radio
  • select (which falls under its own syntax)
  • textarea ('')
  • submit
  • reset
Within theses types there are other options that you must set, these will be discussed below.

Form

All form elements must be placed within the form tags.
The 'action' attribute can then be set to the program within the server that will process that particular form.
The 'method' attribute can also be set, this will be explained later.

Name

Each parameter passed to the server back-end must be discussed within the team so that they match up.
The name of the parameter can be set with the "name" option within the input type. This will be demonstrated by URL link.
For example this webpage will submit a URL like this when filled out.


URL: http://library.whosville/bsearch?author=tonien&year=2000&sub=cs&sub=phy

Value

The value parameter is the value that is sent as a parameter within the html link. While this is not usefull with searchbars, checkboxs/radio/etc can have a shortend value that is sent over. This must also match the backend.

Method

The method attribute can either be set to "get" or "post"

method="get" method="post"
Visibility Data is visibile in the URL Data is not displayed within the URL
History Saved in browser History Not saved and cannot be bookmarked.
This also means that the user will have to re-enter their information
Restrictions URL is limited to a maximum length, therefore you cannot breach that in the parameteres.
Only ASCII Characters are allowed 		No restrictions
